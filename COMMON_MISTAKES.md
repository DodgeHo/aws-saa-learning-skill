# Common Mistakes — 常见易错点汇总（精简）

以下为课程复习中常碰到的错误、误区及快速修正建议。

1. 权限过宽的 IAM 策略：避免使用 `*` 或将管理权限直接授予用户；优先使用最小权限原则与角色（Role）。
2. 将敏感配置硬编码在代码或 S3：使用 `SSM Parameter Store` 或 `Secrets Manager` 并启用加密。 
3. 忽略可用区冗余：在生产架构中不要把资源仅放在单个 AZ，使用跨 AZ 部署以提高可用性。
4. 使用单一 EC2 实例承载关键服务而不做自动伸缩或备份：设计时考虑 ASG/ELB 与故障转移。
5. 未配置 S3 生命周期和加密：对长期归档数据使用合适的存储类与 SSE/KMS。
6. 忽视监控与告警：为关键指标建立 CloudWatch 告警并定期检查日志与指标趋势。
7. 把数据库凭证放在环境变量或代码库：使用 IAM Roles + Secrets Manager，或加密存储凭证。
8. 不正确选择一致性/读写模型：例如对 DynamoDB 未合理设置分区键导致热点问题；为读密集型工作负载使用缓存（ElastiCache）。
9. 网络设计忽略 NACL 与 Security Group 差异：Security Group 有状态、NACL 无状态，按需使用。
10. 依赖默认 VPC/默认安全组做生产部署：创建自定义 VPC 以实现更细粒度网络控制。
