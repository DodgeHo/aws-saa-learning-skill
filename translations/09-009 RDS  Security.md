# RDS 与 Aurora 安全

## 学习目标
- 了解静态/传输加密方式
- 掌握认证与网络控制手段
- 了解审计日志的保留方式

## 重点速览
- 静态加密使用 KMS，需在创建时启用
- 传输加密使用 TLS
- 支持用户名密码与 IAM 认证

## 详细内容
**静态加密**：
- 使用 KMS 加密主库与副本
- 未加密主库无法创建加密只读副本
- 现有未加密库需“快照 -> 恢复”为加密库

**传输加密**：
- 客户端通过 TLS 连接，使用 AWS TLS 根证书

**认证与网络**：
- 支持用户名/密码
- 支持 IAM 认证（如 EC2 角色）
- 安全组控制端口/IP/来源

**审计日志**：
- 启用审计日志并发送到 CloudWatch Logs 以长期保留
