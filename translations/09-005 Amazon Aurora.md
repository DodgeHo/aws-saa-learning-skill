# Amazon Aurora 概览

## 学习目标
- 理解 Aurora 的定位与优势
- 记住高可用存储模型与读扩展方式
- 掌握 writer/reader 端点概念

## 重点速览
- Aurora 是 AWS 专有、兼容 MySQL/PostgreSQL
- 存储自动扩展：10 GB -> 128 TB
- 6 副本跨 3 AZ，写入需 4/6，读取需 3/6

## 详细内容
性能与兼容性：
- MySQL 性能可高约 5 倍，PostgreSQL 约 3 倍（云优化）
- 驱动兼容 MySQL/PostgreSQL

高可用存储：
- 跨 3 AZ 复制 6 份数据
- 自我修复与条带化存储，降低单点风险

读扩展与故障转移：
- 单一 writer 实例写入
- 最多 15 个读副本
- 故障转移通常 < 30 秒
- 读副本支持跨 Region

端点：
- **Writer endpoint**：始终指向主写实例
- **Reader endpoint**：在读副本间负载均衡
