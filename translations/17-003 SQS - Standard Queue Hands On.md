```markdown
**学习目标**

- 掌握在控制台创建 SQS 标准队列时的关键配置项与常用操作（发送/接收、清空、监控）。
- 理解加密、访问策略与死信队列的基础用途与设置位置。

**重点速览**

- 控制台要点：选择队列类型（Standard / FIFO）、设置可见性超时、等待时间（长轮询）、消息保留期与最大消息大小（256 KB）。
- 加密：默认 SSE-SQS，可切换为 KMS CMK；访问控制通过 IAM 与 SQS 资源策略管理；可配置死信队列与重驱动策略。

**详细内容**

- 常见操作：
  - 创建队列：命名并选择类型，配置可见性超时、保留期、延迟、最大消息大小和加密选项。
  - 发送/接收消息：控制台可直接发送“测试消息”，消费者可轮询并查看消息属性与元数据（消息 ID、接收计数等）。
  - 清空队列：使用 Purge 操作删除所有消息（开发时有用，生产慎用）。
  - 监控：通过控制台查看队列深度、最旧消息年龄等 CloudWatch 指标，用于触发 Auto Scaling。 
  - 访问策略：在队列上配置 JSON 资源策略以允许跨账户或服务写入（例如 SNS 将消息写入 SQS）。

**自测问题**

1. 在控制台创建队列时，哪些设置会影响消费者处理行为？
2. Purge 操作与 DeleteMessage 有何不同？
```
