# 高可用与可扩展性

## 学习目标
- 区分高可用与可扩展性
- 理解垂直/水平扩展的差异
- 了解多 AZ 与负载均衡的作用

## 重点速览
- 高可用：故障时仍能服务
- 可扩展：负载增长时可扩容
- 水平扩展 + 负载均衡是 AWS 常用组合

## 详细内容
**高可用（High Availability）**
- 目标：单点故障不影响服务
- 常见做法：多 AZ 部署、冗余架构

**可扩展性（Scalability）**
- 目标：负载增加时保持性能
- 垂直扩展：升级实例规格
- 水平扩展：增加实例数量

AWS 常见组合：
- 多 AZ + 负载均衡 = 高可用
- Auto Scaling + 负载均衡 = 可扩展

## 自测问题
- 高可用与可扩展性的核心差别是什么？
- 水平扩展相比垂直扩展的优点是什么？

水平扩展, 这意味着您可以增加拥有的实例数量,

用AWS的术语来说,

这称为横向扩展或纵向扩展｡

Out, 当您增加执行严修的数目时｡ 

在中, 当您减少执行严修数目时,

这可用于其他扩充群组或负载平衡器｡

最后, 高可用性是指您在多个AZ上运行同一应用程序的同一实例,

因此, 这适用于启用了多AZ的自动扩展器组或启用了多AZ的负载平衡器｡
