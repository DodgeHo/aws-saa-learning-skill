# AWS 控制台导览

## 学习目标
- 熟悉控制台首页的主要区域
- 理解区域选择器与服务范围
- 学会快速查找服务与判断可用性

## 重点速览
- 选择离你最近的区域可获得更低延迟
- 控制台服务可通过“服务”菜单或搜索栏访问
- 部分服务是全球服务，不受区域影响

## 详细内容
进入 AWS 控制台首页后，右上角是**区域选择器**。建议选择距离自己最近的区域以降低延迟。选择区域与地理位置相关，但你不必真的身处该区域。

首页会显示最近访问过的服务、账户健康状况、成本与使用信息等内容。控制台界面可能会随时间变化，但主要逻辑保持一致。

查看服务有两种方式：
- 点击左上角“服务”，按字母或类别浏览。
- 使用搜索栏直接输入服务名（如 Route 53），可以快速跳转服务或相关文档。

以 Route 53 为例，它是**全球服务**，右上角会显示“Global”，无需选择区域。大多数其他服务是**区域服务**（如 EC2），因此资源会随你选择的区域而变化。课程操作过程中建议始终使用同一个区域，避免资源分散。

如果某服务在你所在区域看不到，可到 AWS 全球基础设施页面查看**区域服务可用性表**。必要时切换到提供该服务的区域。
