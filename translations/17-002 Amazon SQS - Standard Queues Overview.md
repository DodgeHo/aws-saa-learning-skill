**学习目标**

- 了解 Amazon SQS 标准队列的核心特性与行为（吞吐、保留、重复与排序保证）。
- 会在架构题中识别何时使用 SQS 解耦服务并与 Auto Scaling 集成。

**重点速览**

- 标准队列：几乎无限吞吐与容量；消息保留期默认 4 天（最长 14 天）；消息最大 256 KB。
- 保证至少一次投递（可能重复）且为尽力而为的顺序（不保证全局有序）。

**详细内容**

- 概念：生产者使用 SendMessage 将消息放入队列；消费者轮询（poll）队列并处理后调用 DeleteMessage 删除消息。消费者可并行扩展以提高处理吞吐。
- 特性与实操要点：
  - 吞吐/容量：无服务器限制，可处理高并发消息流。
  - 保留期：消息在队列中的最长保存时间为 14 天，默认 4 天；超期会丢失。
  - 大小限制：单条消息上限 256 KB（可通过分段或 S3 传递更大负载）。
  - 重复与顺序：标准队列允许重复投递且不保证严格顺序；若需要顺序与去重，应考虑 FIFO 队列。
  - 与 Auto Scaling 集成：通过 CloudWatch 的 ApproximateNumberOfMessagesVisible 指标触发扩缩容，扩展消费者以处理累积队列长度。
  - 安全：传输使用 HTTPS，静态加密可用 SSE-SQS 或 KMS；访问控制可用 IAM 与 SQS 资源策略（跨账户或服务写入）。

2. 如何根据队列长度自动扩展消费者？列出关键指标。
