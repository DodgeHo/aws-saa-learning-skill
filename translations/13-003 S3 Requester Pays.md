# S3 Requester Pays（请求者付费）

## 学习目标
- 理解 Requester Pays 的用途与计费边界
- 知道启用条件与对请求者身份的要求

## 重点速览
- 默认情况下 Bucket 所有者承担存储与数据传输费用
- 启用 Requester Pays 后，下载数据的请求者承担与数据传输相关的费用（存储费仍由桶所有者承担）
- 请求者必须为已认证的 AWS 主体（不能匿名），以便 AWS 能将流量费用计入请求者

## 详细内容
- 适用场景：共享大型数据集给第三方用户或其他账户时，避免桶所有者承担高额下载流量费
- 计费模型：Bucket 保持存储计费不变；读取/下载相关的网络费用由发起下载的请求者承担
- 要求与限制：请求者必须使用 AWS 认证的凭证（IAM 用户/角色/临时凭证）访问，无法对匿名公网访问计费
