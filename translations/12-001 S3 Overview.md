# Amazon S3 概览

## 学习目标
- 了解 S3 的常见用例与设计目标
- 理解 bucket、object、key 的概念与命名规则
- 知道对象大小、multipart 上传与元数据要点

## 重点速览
- S3 是高可用、可无限扩展的对象存储，常用于备份、归档、媒体托管、数据湖、静态网站等
- Bucket 名称必须全局唯一并在特定区域创建；对象由 key（类似路径）标识
- 单个对象最大 5 TB；大于 5 GB 的对象通常使用 multipart 上传；对象可携带元数据与标签

## 详细内容
核心用途：备份/归档（Glacier）、灾备、混合云存储、媒体托管、数据湖与静态网站托管。

Bucket 与命名：
- Bucket 是命名空间的顶层容器，名称在所有 AWS 账户中必须唯一（全局唯一）
- Bucket 在创建时绑定到某个区域（常见误解：S3 看似全局，实际上 Bucket 有区域）
- 命名规则：仅小写字母、数字和连字符，长度 3–63，不可为 IP 格式等

对象（Object）与 key：
- 对象是存储的文件，key 是对象的完整路径（可包含斜杠，控制台会显示为“文件夹”）
- 最大对象大小 5 TB；若大文件需使用 multipart 上传以提升可靠性与并行性
- 对象支持系统/用户元数据、标签与版本 ID（若开启版本控制）

其他注意：S3 与许多 AWS 服务集成，常作为数据持久层或事件触发源（PUT 触发 Lambda、EventBridge 等）。
