# S3 复制（Replication）要点

## 学习目标
- 理解 S3 复制的行为边界与常见设置
- 知道何时使用 S3 批量复制来复制历史对象

## 重点速览
- S3 复制规则默认只复制在规则启用之后产生的新对象
- 若需复制已有对象，使用 S3 Batch Replication（批量复制）
- 可选地复制删除标记；永久删除的特定版本通常不会被复制（可防止恶意或意外永久删除跨桶传播）
- S3 复制不形成“复制链”：源桶→目标桶→再目标桶 不会级联

## 详细内容
- 新对象复制：启用复制后，后续的新 PUT/复制会触发跨桶复制
- 复制现有对象：需显式运行 S3 批处理来处理已存在的对象或失败的复制
- 删除行为：默认不复制删除标记；可在规则中启用删除标记复制（视业务需求）
- 复制链限制：复制不会自动穿透多跳目标，设计多级复制时需考虑此限制

## 自测问题
- 启用复制后为什么有时目标桶还没有对象？
- 如何将已有对象从源桶复制到目标桶？
