# 安全组与常见端口

## 学习目标
- 理解安全组的作用与规则特性
- 识别超时与拒绝连接的常见原因
- 记住考试常见端口

## 重点速览
- 安全组只包含“允许”规则
- 默认：入站拒绝、出站允许
- 超时通常是安全组问题；连接被拒绝通常是应用问题

## 详细内容
安全组是 EC2 实例外部的防火墙，控制进出实例的流量。规则以协议、端口、来源（CIDR 或安全组）为核心。安全组可以：
- 绑定多个实例
- 一个实例绑定多个安全组
- 仅在同一 Region/VPC 内生效

如果访问应用一直“超时”，通常是安全组未放行端口；如果是“连接被拒绝”，说明安全组已放行但应用未启动或端口未监听。

高级用法：安全组可以相互引用。这样可以允许“某个安全组里的实例”访问另一组实例，而无需固定 IP，常用于负载均衡场景。

## 常见端口（考试重点）
- **22**：SSH（Linux）/ SFTP
- **21**：FTP
- **80**：HTTP
- **443**：HTTPS
- **3389**：RDP（Windows）
