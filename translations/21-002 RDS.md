## 学习目标

- 理解 Amazon RDS 的定位：托管关系型数据库（如 PostgreSQL、MySQL、Oracle、SQL Server 等）。
- 掌握 RDS 的关键特性：实例预配、存储配置、读副本、Multi-AZ、备份与恢复、维护窗口与安全机制（IAM、KMS、网络）。

## 重点速览

- RDS 是托管关系型数据库服务，需要预配实例和 EBS 存储；适合 OLTP 与需要 SQL/事务功能的场景。
- 支持读副本扩展读取、Multi-AZ 提供灾备、自动备份与时间点恢复（最多 35 天）、手动快照用于长期保留。
- 集成 RDS Proxy 与 Secrets Manager、支持 IAM 认证，网络层使用安全组，静态与传输层支持加密（KMS、SSL/TLS）。

## 详细内容

- 架构与运维：
	- 需为实例预配计算资源与 EBS 类型/容量；存储层有自动扩缩但实例仍需调配。 
	- 多可用区（Multi-AZ）用于高可用，故障时自动故障切换，但 Multi-AZ 备用不可用于读查询。 

- 扩展与性能：
	- 使用只读副本来扩展读取吞吐。 
	- 可结合 RDS Proxy 以提高连接管理与缩短冷启动延迟。 

- 备份与恢复：
	- 自动备份支持时间点恢复（默认最多 35 天）；手动快照用于长期保留与迁移。 
	- 维护窗口会触发补丁与升级，可能带来短暂停机。 

- 安全与凭据管理：
	- 支持用户名/密码与 IAM 身份验证，推荐与 Secrets Manager 集成管理凭据；使用 KMS 进行静态加密，使用 SSL/TLS 保护传输。
