**学习目标**

- 理解 SQS FIFO 队列提供的顺序与去重保证，以及其性能限制与使用约束。

**重点速览**

- FIFO（先进先出）保证消息按顺序交付并支持基于内容的重复数据删除；吞吐相对受限（无批处理时约 300 TPS，批处理可达更高）。
- FIFO 队列名称需以 `.fifo` 结尾，使用消息组 ID 来并行化不同组的消息处理。

**详细内容**

- 顺序保证：在同一消息组（Message Group ID）内，消息按发送顺序被消费者接收与处理；不同消息组可并行处理以提高吞吐。
- 去重：FIFO 支持去重功能（content-based deduplication 或显式去重 ID），在短时间窗口内重复发送的相同消息可被去重。
- 吞吐与限制：由于顺序约束，吞吐低于标准队列；需评估是否能接受较低吞吐以换取顺序与去重保证。
- 使用场景：金融交易、订单处理等对顺序与一次性处理有强要求的场景适用 FIFO。

**自测问题**

1. FIFO 队列如何保证消息顺序？如何同时提升吞吐？
2. 创建 FIFO 队列时有哪些命名与配置要求？
