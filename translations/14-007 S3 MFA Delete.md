教师：现在, 我们来讨论一项名为MFA

Delete的安全功能｡

MFA代表多因素身份验证, 这是一种强制用户在设备上生成代码的方法｡

例如, 它可以是具有Google身份验证器应用程序或其他应用程序的移动电话,

或者是另一个应用程序, 或者是诸如MFA硬件设备之类的硬件｡
## 学习目标

- 理解 S3 的 MFA Delete 功能是什么以及它的安全意义。
- 知道在何种操作下需要 MFA（例如永久删除对象版本），以及启用/禁用 MFA Delete 的前提条件。

## 重点速览

- MFA Delete 是针对破坏性操作（例如永久删除对象版本或关闭版本控制）增加的一层保护，需要多因素验证码。
- 只有在存储桶启用了版本控制后才可使用 MFA Delete；启用/禁用 MFA Delete 必须由 root 账户执行（通过 CLI）。

## 详细内容

1. 什么是 MFA Delete
- MFA Delete 要求在执行高度破坏性的 S3 操作（如永久删除某个版本）时提供多因素验证码，从而降低误删或被滥用的风险。

2. 何时需要 MFA
- 常见需要 MFA 的场景：永久删除对象版本、在某些情况下更改版本控制相关的关键设置。
- 常规列出版本或启用/查看版本控制并不需要 MFA。

3. 启用条件与限制
- 要使用 MFA Delete，存储桶必须启用版本控制。
- 启用或禁用 MFA Delete 需使用 root 账户并通过 AWS CLI 执行（控制台目前无法直接切换）。
