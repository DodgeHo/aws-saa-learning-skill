```markdown
**学习目标**

- 掌握在控制台选择与部署不同 Storage Gateway 类型时的关键配置点与决策要点。
- 能区分在考试中常见选项（部署平台、缓存策略、存储类、身份验证）并快速选择正确答案。

**重点速览**

- 控制台创建步骤包括选择网关类型（S3 文件、FSx 文件、卷、磁带）、部署平台（VM/硬件/EC2）、以及缓存与授权配置。
- 卷网关的两种模式（cached / stored）与磁带网关的 VTL 工作流程是重要的考试区分点。

**详细内容**

- 常见控制台选项：
  - 平台：VMware / Hyper-V / KVM / EC2 / 硬件设备（适用于无虚拟化环境）。
  - S3 文件网关：配置目标 S3 桶、NFS/SMB 协议、IAM 角色与可选的 AD 集成（SMB 时）。
  - FSx 文件网关：用于本地缓存 FSx for Windows 文件服务器数据，适合需要 SMB 与低延迟访问的场景。
  - 卷网关：选择 cached（本地缓存热数据）或 stored（本地保存全部数据并备份到 S3），并设置备份频率与 EBS 快照策略。
  - 磁带网关：配置 VTL、备份软件连接与归档到 Glacier 的生命周期策略。
- 实操提示（考试视角）：不要在控制台选项上过度细节化，记住关键差异（协议、缓存位置、目标存储类型、是否支持 AD）。

**自测问题**

1. 在没有虚拟化基础设施的分支机构，应如何部署 Storage Gateway？
2. 创建 S3 文件网关时，需要注意哪些身份验证或权限设置？
```
