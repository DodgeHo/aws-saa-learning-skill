# IAM 角色实操

## 学习目标
- 在 IAM 中创建服务角色
- 为角色附加合适策略
- 理解“受信任实体”含义

## 重点速览
- 角色用于 AWS 服务，不是给人登录用的
- 创建角色时必须选择受信任服务（如 EC2）
- 权限通过策略附加到角色

## 详细内容
进入 IAM 的“角色”页面，可以看到已有角色。这里创建一个新角色，选择 **AWS 服务**作为受信任实体，并指定服务为 **EC2**。

接着为角色附加策略，例如 `IAMReadOnlyAccess`，用于演示 EC2 实例读取 IAM 信息。然后为角色命名（如 `DemoRoleForEC2`），确认受信任实体为 EC2 并创建。

创建完成后，角色会出现在列表中，可查看其权限策略。该角色会在后续 EC2 章节中实际使用。
