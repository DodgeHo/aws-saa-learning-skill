# 安全组实操

## 学习目标
- 查看并编辑安全组入站/出站规则
- 理解端口规则对访问的影响
- 掌握超时问题的排查思路

## 重点速览
- 删除 HTTP(80) 规则会导致网页访问超时
- 使用“我的 IP”要注意 IP 变化
- 一个实例可绑定多个安全组

## 详细内容
进入 EC2 实例的“安全性”标签可快速查看安全组信息。更完整的管理入口在左侧“网络和安全 → 安全组”。

本例中入站规则包含：
- SSH(22) 允许访问
- HTTP(80) 允许访问

删除 HTTP 规则后，访问网页会出现超时，说明安全组阻止了流量。重新添加 HTTP(80) 规则即可恢复访问。

入站规则可指定端口与来源（CIDR、安全组、前缀列表等）。如果只允许“我的 IP”，一旦 IP 变化就会无法访问。

出站规则默认允许所有 IPv4 流量，保证实例可访问互联网。

安全组之间是多对多关系：一个实例可挂多个安全组，一个安全组可用于多个实例。
