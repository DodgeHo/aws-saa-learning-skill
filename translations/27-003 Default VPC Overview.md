## 学习目标

- 理解默认 VPC 的特点、何时使用以及与自定义 VPC 的区别。

## 重点速览

- 默认 VPC 是 AWS 帐户在每个区域自动创建的、能马上使用的网络环境；带默认子网、路由与安全组，可直接启动实例并分配公网 IP。
- 自定义 VPC 提供更多控制（CIDR、子网划分、路由表、网络 ACL 等）。

## 详细内容

1. 默认 VPC 的优点：开箱可用、便于测试与快速启动实例。
2. 局限性：控制较少、CIDR 固定，长期生产环境通常建议创建自定义 VPC 以便设计子网与安全边界。
3. 默认 VPC 与安全：默认安全组有宽松入站规则（允许同组流量），需在生产中收紧策略。
