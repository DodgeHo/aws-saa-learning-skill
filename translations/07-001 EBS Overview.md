# EBS 概览

## 学习目标
- 理解 EBS 的作用与特性
- 掌握 EBS 与 AZ 绑定的限制
- 了解“终止时删除”行为

## 重点速览
- EBS 是网络块存储，可在实例运行时挂载
- 一次只能挂载到一个实例（常见场景）
- 卷绑定到 AZ，跨 AZ 需快照迁移

## 详细内容
EBS（Elastic Block Store）是 EC2 的网络块存储。实例终止后，若卷未删除，数据可保留并挂载到新实例。

关键特性：
- **网络驱动器**：可快速挂载/卸载，但存在网络延迟
- **AZ 绑定**：卷只能在创建的 AZ 内使用
- **容量预配置**：需提前指定容量与 IOPS

**终止时删除（Delete on termination）**：
- 根卷默认删除
- 额外挂载卷默认保留
- 可手动修改，以保留根卷数据（考试常见场景）
