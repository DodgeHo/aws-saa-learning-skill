## 学习目标

- 理解 AWS 中事件驱动架构的基本概念与常用服务（EventBridge、SNS、SQS、Lambda）。
- 能够为常见场景设计事件流与选择合适的服务组合。

## 重点速览

- 事件来源（应用、服务、S3、CloudWatch 等）通过 EventBridge 或 SNS 分发到消费者（Lambda、SQS、Kinesis 等）。
- SQS 提供可靠的队列缓冲，SNS 适合广播，EventBridge 支持事件路由与规则匹配。

## 详细内容

1. EventBridge：支持模式匹配规则、跨账户/跨区域事件总线，适合复杂事件路由与集成 SaaS 事件源。
2. SNS：发布/订阅模型，低延迟广播到多个订阅者；可与 SQS 结合以保证可靠消费。
3. SQS：可见性超时、死信队列（DLQ）等机制帮助处理失败与重试；标准队列与 FIFO 队列适配不同一致性需求。
4. Lambda 常作为无服务器消费者执行事件处理逻辑；高吞吐场景应关注并发限制、批处理与错误处理策略。

## 自测问题

1. 比较 SNS 与 SQS 的主要差异与典型使用场景。
2. 描述如何使用 EventBridge 将 S3 上传事件转发给多个不同服务并保证至少一次处理。
