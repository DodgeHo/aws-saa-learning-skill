# IAM 小结

## 学习目标
- 回顾 IAM 的核心组件与安全实践
- 建立对控制台/CLI/SDK 访问方式的整体理解

## 重点速览
- 用户映射真实人员，组用于权限管理
- 策略是 JSON 权限文档，可附加到用户/组/角色
- 角色用于 AWS 服务访问
- MFA 与密码策略提升账号安全

## 详细内容
本节回顾 IAM 的核心内容：

- **用户**：映射到组织内的实际人员，拥有控制台登录凭据。
- **组**：只包含用户，用于集中管理权限。
- **策略**：JSON 权限文档，可附加到用户、组或角色。
- **角色**：给 AWS 服务使用（如 EC2、Lambda），让服务代表你执行操作。
- **安全强化**：启用 MFA，并设置密码策略。

访问方式方面，你可以通过控制台操作，也可以使用 **CLI** 或 **SDK**。CLI/SDK 需要访问密钥进行身份验证。审计与治理可通过**凭据报告**与 **Access Advisor** 完成。
