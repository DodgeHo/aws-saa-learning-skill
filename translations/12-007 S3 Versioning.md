# S3 版本控制（Versioning）概览

## 学习目标
- 理解 S3 版本控制的作用与启用位置
- 掌握版本带来的恢复与回滚能力

## 重点速览
- 版本控制在 Bucket 级别启用；对同一 key 的每次上传都会创建新版本
- 版本能防止意外删除（删除会生成删除标记），支持回滚到旧版本
- 暂停版本控制不会删除已有版本；未启用时的对象版本 ID 为 null

## 详细内容
- 启用：在 Bucket 的 `Properties` 开启 Versioning
- 行为：每次上传同一 key 时生成新版本 ID；旧版本保留
- 删除：对对象的常规删除会添加“删除标记”，底层版本仍存在；永久删除需删除特定版本 ID
- 恢复：可删除删除标记或直接复制旧版本到当前 key 实现回滚
