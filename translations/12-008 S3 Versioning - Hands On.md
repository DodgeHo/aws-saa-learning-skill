# S3 版本控制 - 实操要点

## 学习目标
- 能在控制台启用版本控制并观察版本行为
- 掌握删除标记与永久删除的区别与恢复方法

## 重点速览
- 启用版本控制后，后续上传会产生新版本 ID；启用前上传的对象版本 ID 为 null
- 普通删除会添加删除标记（可恢复）；删除特定版本 ID 才是永久删除且不可逆

## 实操步骤与示例流程
1. 在 Bucket 的 `Properties` 启用 Versioning
2. 上传并覆盖同一 key（例如 `index.html`），观察控制台中出现的新版本 ID
3. 在对象视图启用 `Show versions`，查看每个版本及其版本 ID
4. 恢复流程：若对象被删除（出现删除标记），删除该删除标记即可恢复前一版本；若永久删除某个版本，需谨慎操作
