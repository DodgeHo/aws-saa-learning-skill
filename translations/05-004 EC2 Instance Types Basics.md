# EC2 实例类型基础

## 学习目标
- 认识 EC2 实例类型的分类与命名规则
- 理解通用型、计算型、内存型、存储型的区别
- 知道如何查找实例类型与价格

## 重点速览
- 命名结构：`m5.2xlarge` = 类型 + 代际 + 规格
- 通用型适合多数工作负载，课程使用 `t2.micro`
- 计算/内存/存储优化面向特定场景

## 详细内容
EC2 实例按用途分为多类：
- **通用型（General Purpose）**：计算/内存/网络均衡，适合 Web 服务器、代码仓库等。
- **计算优化（Compute Optimized）**：高 CPU，适合批处理、媒体转码、HPC、机器学习、游戏服务器等。
- **内存优化（Memory Optimized）**：高内存，适合内存数据库、缓存、BI 分析、实时处理等。
- **存储优化（Storage Optimized）**：高本地存储性能，适合 OLTP、分布式文件系统等。

命名示例 `m5.2xlarge`：
- `m`：实例家族（通用型）
- `5`：代际
- `2xlarge`：规格大小（越大 CPU/内存越多）

本课程使用免费层的 `t2.micro`（每月 750 小时）。如果想比较实例规格与价格，可查看官方实例类型页面或第三方汇总站点（如 ec2instances.info）。
