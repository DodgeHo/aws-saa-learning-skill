**学习目标**

- 了解 Amazon MQ 的定位：托管的消息代理（支持 ActiveMQ、RabbitMQ），适用于迁移使用开放协议的传统消息应用。
- 理解与 SQS/SNS 的区别、可用性与持久化存储（EFS）配置要点。

**重点速览**

- Amazon MQ 提供对传统协议（AMQP、MQTT、STOMP、OpenWire 等）的托管支持，便于无需改造客户端即可上云。
- 与 SQS/SNS 相比，Amazon MQ 更像传统代理，需要关注实例/代理的可用性与运维（可以启用多 AZ 高可用）。

**详细内容**

- 关键要点：
  - 使用场景：当现有应用依赖开放标准消息协议且不希望重写为 SQS/SNS API 时，选择 Amazon MQ（托管 ActiveMQ/RabbitMQ）。
  - 可用性：推荐多 AZ 部署以提高可用性；Amazon MQ 可配置 EFS 作为后端持久存储以支持故障切换与数据持久化。
  - 扩展性与运维：相比 SQS/SNS 的无限扩展，Amazon MQ 的可扩展性受代理资源限制（需要考虑实例规格与集群配置），并可能需要更多运维关注。
  - 安全与集成：支持常见代理认证/加密机制，客户端通过开放协议接入，适合与传统系统对接。
