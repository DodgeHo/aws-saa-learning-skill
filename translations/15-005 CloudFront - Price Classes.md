## 学习目标

- 理解 CloudFront 的价格分类（Price Classes）概念及其对成本与性能的权衡。
- 会在分发配置中选择合适的 Price Class 以控制可用边缘位置范围与费用。

## 重点速览

- CloudFront 在不同地理区域的边缘位置数据传输成本不同；Price Class 允许限制使用的边缘区域以降低成本。
- 常见选项为 Price Class 100、200 与 All（全部），从覆盖面最小到最大，覆盖范围越大性能越好但成本越高。

## 详细内容

1. 为什么有价格分类
- 不同地区的 egress 成本差异会影响总费用；Price Class 通过限制边缘位置来控制流量发送到较低成本区域，从而降低费用但可能牺牲少数区域的性能。

2. 各类含义
- Price Class 100：仅覆盖最便宜的区域（如北美与欧洲等）。
- Price Class 200：覆盖更多区域，但仍排除部分最昂贵区域。
- Price Class All：覆盖全球所有边缘位置，提供最佳延迟表现，但成本最高。

3. 选择建议
- 若目标用户主要集中在少数地区，可选择较小的 Price Class 以节省成本；若需要全球最佳性能且预算允许，选择 All。
