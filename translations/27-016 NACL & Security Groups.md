## 学习目标

- 熟悉 NACL 与 Security Group 的差异、默认行为及在 VPC 中的协同使用场景。

## 重点速览

- Security Group：有状态、防火墙级别在实例层面；NACL：无状态、在子网层面，规则按照先允许后拒绝或反向评估。

## 详细内容

1. Security Group 是有状态的，允许入站后自动允许相应出站响应；NACL 是无状态，入/出站规则需分别配置。
2. 推荐实践：把细粒度的实例访问控制放在 Security Group，使用 NACL 作为额外的子网级别保护或在合规场景中使用。
