# IAM 入门：用户、组与策略

## 学习目标
- 理解 IAM 的作用与全球性特点
- 区分用户、组与策略的关系
- 掌握最小权限原则的意义

## 重点速览
- Root 用户只用于初始化配置，不应日常使用
- 组只能包含用户，不能嵌套组
- 权限通过策略（JSON）赋予用户或组

## 详细内容
IAM（Identity and Access Management）是 AWS 的身份与访问管理服务，属于**全球服务**。我们创建账号时已经隐式使用了 IAM，因为默认会创建 root 用户。root 用户只应在初始化设置时使用，不应日常登录或共享。

正确做法是为组织成员创建**IAM 用户**。用户可以加入一个或多个**组**，组用于集中管理权限。组只能包含用户，不能包含其他组；用户也可以不属于任何组，但这不是最佳实践。

举例：如果 Alice、Bob、Charles 是开发人员，可将他们加入“developers”组；David 与 Edward 属于运维组；若 Charles 与 David 同时属于审计组，他们就会同时继承开发/运维组与审计组的权限。

权限通过**策略（Policy）**授予。策略是 JSON 文档，用于描述允许或拒绝哪些操作。例如允许使用 EC2、ELB、CloudWatch 等服务。你不需要是程序员，只要理解策略表达的是“允许谁对什么资源做什么”。

AWS 强调**最小权限原则**：只授予用户完成工作所需的最低权限，避免误操作或安全风险。
