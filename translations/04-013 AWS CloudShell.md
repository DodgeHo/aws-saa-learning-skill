# AWS CloudShell 概览

## 学习目标
- 了解 CloudShell 的用途与限制
- 掌握 CloudShell 的核心功能
- 知道何时使用本地终端替代

## 重点速览
- CloudShell 是 AWS 控制台内置终端，免费使用
- 仅在部分区域可用，需先确认可用性
- 支持持久化存储、上传/下载与多标签页

## 详细内容
CloudShell 是 AWS 控制台右上角的终端入口。它并非所有区域都提供，因此要先确认当前区域是否支持。建议课程中尽量使用同一个区域，以保证 CloudShell 可用。

CloudShell 内置 AWS CLI（通常为 v2），直接使用当前登录账号的凭证和区域。你可以立即执行 AWS CLI 命令，例如列出 IAM 用户。默认区域即当前控制台区域。

它还提供以下便利功能：
- **持久化存储**：在 CloudShell 中创建的文件会保存下来，重启后仍可使用。
- **可配置界面**：字体大小、主题（明暗）可调。
- **文件上传/下载**：可以在控制台中上传本地文件或下载 CloudShell 文件。
- **多标签与分屏**：支持多个终端标签或分列显示。

如果 CloudShell 不可用或不符合你的习惯，也可以继续使用本地终端，功能等价。
