## 学习目标

- 会在控制台创建一个简单的 Global Accelerator（包含监听器、端点组与端点），并验证健康检查与流量路由行为。
- 理解如何为不同区域配置端点组与权重以实现流量控制与故障切换。

## 重点速览

- Hands-on 流程要点：在多个区域启动后端实例或 ALB → 创建 Global Accelerator 并配置监听器（例如 TCP 80）→ 添加端点组（按区域）并将实例/ALB 作为端点添加 → 配置权重与健康检查 → 部署并验证路由与故障切换行为。
- Global Accelerator 会为分发分配两个静态任播 IP，用以全局访问与 DDoS 缓解集成。

## 详细步骤要点

1. 准备后端
- 在两个或多个区域启动后端实例或准备 ALB（实例可返回可识别的内容以便验证来自哪个区域）。

2. 创建 Accelerator
- 在 Global Accelerator 控制台创建加速器，添加监听器（选择协议/端口），并为每个目标区域创建端点组，设置权重与健康检查参数。

3. 添加端点并部署
- 将实例或 ALB 添加为端点，提交创建后等待部署并观察静态任播 IP 与加速器 DNS。

4. 验证与故障切换
- 通过访问加速器的任播 IP/DNS 并观察响应可判断流量到达的区域；通过使某区域不健康（调整安全组）可验证健康检查触发与自动故障切换。

5. 清理与成本注意
- Global Accelerator 会产生成本；若仅为实验请在完成后删除加速器与测试资源以避免费用。
