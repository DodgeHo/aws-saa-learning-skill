## 学习目标

- 了解 Amazon EMR 的定位、常见处理框架（Hadoop、Spark、Presto/Trino）和典型数据处理场景。 
- 掌握 EMR 的集群类型、成本控制（Spot、自动缩放）以及与 S3/Glue 的集成方式。 

## 重点速览

- EMR 是托管的大数据处理平台，适用于批处理、ETL、交互式分析与机器学习训练。 
- 集群由主节点、核心节点与任务节点组成；数据通常存放在 S3（通过 EMRFS 访问）。 

## 详细内容

- 架构与执行：
  - EMR 可运行 Spark、Hadoop、Presto/Trino 等引擎，支持分布式作业并行执行。 
  - 主节点负责协调，核心节点保存 HDFS 元数据或执行数据处理，任务节点用于短期计算任务。 

- 数据与集成：
  - 推荐将持久数据放在 S3，通过 `COPY`、EMRFS 或 Glue Catalog 进行元数据管理与读取。 
  - 常与 AWS Glue、Step Functions、CloudWatch、IAM 等整合构建生产化 ETL 管道。 

- 成本与运维：
  - 使用 Spot 实例与自动缩放降低成本；保持数据在 S3 以避免长期运行集群的存储成本。 
  - 监控作业、日志、指标与合理选择实例类型以优化性能与费用。
