# MFA 与密码策略

## 学习目标
- 了解密码策略的关键选项
- 理解 MFA 的工作原理与价值
- 认识 AWS 支持的 MFA 设备类型

## 重点速览
- 密码策略可规定长度、复杂度与过期规则
- MFA 将“你知道的”和“你拥有的”结合起来
- 重点保护 root 用户与所有 IAM 用户

## 详细内容
在创建用户与组之后，需要进一步保护账号安全。第一道防线是**密码策略**，可设置最小长度、字符类型要求（大小写、数字、特殊字符）、是否允许用户修改密码、密码过期周期（例如 90 天）以及禁止重复使用旧密码。这有助于抵御暴力破解。

第二道防线是**多因素认证（MFA）**。即使密码泄露，攻击者仍需获得你的物理设备才能登录。MFA 组合“你知道的（密码）”与“你拥有的（设备）”，显著提升安全性。至少应为 root 用户启用 MFA，并尽量覆盖所有 IAM 用户。

AWS 常见 MFA 设备类型：
- **虚拟 MFA**：如 Google Authenticator（单设备）、Authy（多设备同步）。
- **U2F 安全密钥**：如 YubiKey（第三方硬件），可用于多个用户。
- **硬件密钥卡**：例如 Gemalto（第三方提供）。
- **GovCloud 专用设备**：如 SurePassID（用于 AWS GovCloud）。
