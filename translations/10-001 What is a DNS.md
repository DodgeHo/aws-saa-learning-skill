# DNS 是什么

## 学习目标
- 理解 DNS 的作用与分层结构
- 认识常见 DNS 术语
- 掌握基本解析流程

## 重点速览
- DNS 将域名解析为 IP
- 分层结构：根 -> TLD -> 二级域 -> 子域
- 解析结果会被缓存（TTL）

## 详细内容
DNS（Domain Name System）负责把人类可读的域名转换为 IP 地址，浏览器再用 IP 访问服务器。

常见术语：
- **域名注册商**：注册域名的机构（如 Route 53、GoDaddy）
- **DNS 记录**：A/AAAA/CNAME/NS 等
- **名称服务器**：负责解析查询的服务器
- **顶级域**：`.com`、`.org` 等
- **二级域**：`example.com`
- **子域**：`api.example.com`

解析流程（简化）：
1. 浏览器查询本地 DNS 解析器
2. 解析器问根服务器获取 TLD 服务器
3. 再问 TLD 获取权威名称服务器
4. 权威服务器返回 A 记录（IP）
5. 解析器缓存结果并返回给浏览器
