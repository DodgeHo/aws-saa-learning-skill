**学习目标**
- 理解 Lambda SnapStart 的作用与适用场景（主要针对 Java）

**重点速览**
- SnapStart 通过对已初始化函数的内存和磁盘状态拍摄快照，加速冷启动时间
- 主要优化 Java 等初始化慢的运行时，可显著降低延迟并提升性能

**详细内容**
Lambda SnapStart 工作流程：在发布新版本时，Lambda 会初始化函数并对其初始化后的内存与磁盘状态创建快照（snapshot）；后续调用直接从该快照恢复，跳过完整的初始化过程，从而大幅减少冷启动时间。当前 SnapStart 主要针对在 Java 11+ 上运行且初始化开销大的函数，适合对延迟敏感的生产工作负载。

注意事项：启用 SnapStart 会改变发布/版本化流程（快照在版本发布时生成），请在使用前测试状态与副作用（例如与外部连接或临时文件相关的初始化逻辑）。
