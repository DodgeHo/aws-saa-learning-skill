## 学习目标

- 理解 S3 Access Points 的目的：简化大规模数据集上的访问控制与隔离。
- 会为不同用例创建访问点并为每个访问点配置单独的访问策略（可面向前缀、只读/读写等）。

## 重点速览

- Access Point 是指向同一桶的逻辑入口点，每个入口点有独立的策略与 DNS 名称，用于按角色/用途分离访问权限。
- 支持公网访问或作为 VPC 私有访问点；VPC 访问需通过 VPC Endpoint 并在端点策略中允许访问目标接入点及桶。

## 详细内容

1. 为什么使用 Access Points
- 当一个桶包含大量不同前缀和用户组时，单一复杂桶策略会难以维护。Access Points 允许为每个使用场景定义更简单、可控的策略（例如 `finance/*`、`sales/*`）。

2. 管理模型
- 每个 Access Point 有自己的策略（类似桶策略），可授予对特定前缀的权限（读/写）。可以创建多个访问点（如 finance、sales、analytics），分别绑定各自的策略。

3. DNS 与访问方式
- 每个接入点有专属 DNS 名称，客户端通过接入点的 DNS 访问 S3。接入点可配置为公共（Internet）访问或仅限 VPC 源。

4. VPC 原点与 VPC Endpoint
- 若接入点配置为 VPC 原点，需在 VPC 中创建 S3 VPC Endpoint，并在端点策略中允许访问目标接入点与桶，从而实现不经 Internet 的私有流量访问。
