## 学习目标

- 理解 NAT Gateway 的工作原理、优缺点及在 VPC 中的使用场景。

## 重点速览

- NAT Gateway 是 AWS 托管的出站网络地址转换服务，具备高可用性、自动扩展，但按流量计费。

## 详细内容

1. NAT Gateway 部署在公有子网并分配 EIP；私有子网路由表把 `0.0.0.0/0` 指向 NAT Gateway 实现出站访问。
2. 优点：托管、可扩展、无需维护；缺点：成本高于 NAT 实例，跨 AZ 需在每个 AZ 部署一个 NAT Gateway 以实现高可用。

1. 描述部署 NAT Gateway 时如何保证跨 AZ 的高可用性。
2. 与 NAT 实例相比，费用和运维上有什么权衡？
