## 学习目标

- 理解 AWS Well-Architected Framework 的五大支柱及如何使用 Well-Architected Tool 评估架构。

## 重点速览

- 五大支柱：可靠性（Reliability）、安全性（Security）、性能效率（Performance Efficiency）、成本优化（Cost Optimization）与运维卓越（Operational Excellence）。
- Well-Architected Tool 提供检查清单、改进建议与风险优先级，有助于发现设计缺陷。

## 详细内容

1. 使用流程：定义工作负载 -> 运行 Well-Architected Tool 问卷 -> 获取改进建议 -> 制定改进计划并跟踪修复。
2. 常见问题：工具提供的是建议而非强制；改进优先级基于风险与业务影响，需要结合实际约束制定可执行计划。

## 自测问题

1. 列出 Well-Architected 的五大支柱并举例说明每项在系统设计中的体现。
2. 描述如何将 Tool 的高风险项纳入产品迭代计划中。
