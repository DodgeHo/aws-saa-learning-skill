# IAM 角色：给 AWS 服务授权

## 学习目标
- 理解 IAM 角色的用途与适用对象
- 区分“人用的用户”与“服务用的角色”
- 认识常见服务角色场景

## 重点速览
- 角色用于 AWS 服务代表你执行操作
- 角色像用户一样需要策略授权
- 常见角色：EC2、Lambda、CloudFormation

## 详细内容
IAM 的最后一个核心组件是**角色（Role）**。当某个 AWS 服务需要代表你执行操作时，它必须拥有权限。与用户类似，角色也需要附加策略，但角色不是给人用的，而是给服务用的。

例如 EC2 实例可能需要访问 S3 或 CloudWatch。此时就需要创建一个 IAM 角色并附加相应权限，然后将角色绑定到 EC2 实例。实例在调用 AWS API 时，会自动使用该角色的权限。

常见的服务角色包括 EC2 实例角色、Lambda 角色、CloudFormation 角色等。下一节我们会创建一个角色并在后续课程使用。
