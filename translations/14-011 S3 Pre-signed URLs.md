## 学习目标

- 理解 S3 预签名 URL 的作用、有效期限制以及权限继承的含义。
- 会在合适的场景生成临时可访问的 URL（用于下载或临时上传），而无需公开整个桶。

## 重点速览

- 预签名 URL 是由拥有访问权限的用户通过控制台/CLI/SDK 生成的带签名 URL，允许持有 URL 的人在有限时间内访问对象。
- 控制台生成的预签名 URL 最长可到 12 小时；CLI（SDK）生成的预签名 URL 最长可到 168 小时（依据 SDK/工具与参数）。
- 使用预签名 URL 的用户在访问该对象时会继承生成 URL 的用户的权限（仅对该对象的临时授权）。

## 详细内容

1. 生成与生效
- 可通过 AWS 控制台、AWS CLI 或 SDK 生成预签名 URL。URL 中包含签名与到期时间，过期后无法再使用。

2. 权限继承
- 生成 URL 的账户/身份必须对对象有相应权限；持有预签名 URL 的用户在有效期内会按生成者的权限被授权访问该对象（例如 GET 或 PUT）。

3. 常见用例
- 私有对象的临时下载（例如仅限登录用户下载付费视频）。
- 临时允许外部用户将文件上传到指定位置（POST/PUT），而不将桶设为公开。

4. 注意事项
- 控制台/CLI/SDK 的默认或最大到期时间可能不同；在生成时确认到期时长是否满足业务需求。
