## 学习目标

- 理解 AWS Database Migration Service（DMS）的用途、迁移类型与基本流程。
- 掌握 DMS 在异构数据库迁移与同构迁移中的常见配置要点与限制。

## 重点速览

- DMS 用于将数据从源数据库迁移到目标数据库，支持全量迁移与持续复制（CDC）。
- 常见场景：本地到 RDS/Aurora、RDS 到 Aurora、数据库版本升级或跨引擎迁移（异构）。

## 详细内容

1. 架构要点：DMS 包括复制实例（Replication Instance）、源/目标端点与迁移任务；复制实例执行提取、转换与加载操作。
2. 迁移模式：全量加载（Full load）、全量+持续复制（Full load + CDC）、仅 CDC。选择取决于停机窗口与数据一致性要求。
3. 限制与注意：某些对象（如触发器、存储过程）可能不会被迁移，需使用 schema 转换工具（AWS SCT）做异构迁移的模式转换。
