# AWS 云概览：区域与可用区

## 学习目标
- 了解 AWS 的发展与全球规模
- 理解 Region 与 Availability Zone 的概念
- 掌握选择区域的主要因素

## 重点速览
- Region 是地理区域，AZ 是区域内相互隔离的数据中心集群
- 选择区域要考虑合规、延迟、服务可用性与价格
- AWS 同时提供区域级服务和全球级服务

## 详细内容
AWS 最初于 2002 年在亚马逊内部使用，2004 年公开发布 SQS，2006 年扩展并推出 SQS、S3 与 EC2。如今大量应用运行在 AWS 上，例如 Dropbox、Netflix、Airbnb 以及 NASA。

从行业地位来看，AWS 在云计算市场长期处于领导者位置，并拥有庞大的活跃用户规模。通过学习 AWS，你是在学习被广泛使用的云平台能力。

在 AWS 上可以构建几乎所有类型的应用：企业 IT 上云、备份与存储、大数据分析、网站托管、移动/社交应用后端，甚至游戏服务器。云的使用场景几乎没有上限。

AWS 是全球性的，全球基础设施包括：
- **Region（区域）**：分布在全球的地理区域（例如 `us-east-1`、`eu-west-3`）。大多数服务是区域级的，资源通常只在创建它的区域内可见。
- **Availability Zone（可用区）**：区域内的一个或多个数据中心集群，通常一个区域包含 3~6 个 AZ。AZ 之间相互隔离，以防止单点故障。
- **Edge Locations / Points of Presence（边缘位置/存在点）**：用于低延迟内容分发的全球节点。

选择部署区域时常见考虑因素：
1. **合规**：数据是否需要留在特定国家/地区。
2. **延迟**：离用户更近通常延迟更低。
3. **服务可用性**：某些服务并非所有区域都有。
4. **价格**：不同区域价格不同。

此外，AWS 还存在**全球服务**（如 IAM、Route 53、CloudFront、WAF）和**区域服务**（如 EC2、Elastic Beanstalk、Lambda、Rekognition）。判断某服务在某区域是否可用，可通过官方区域服务表查询。
