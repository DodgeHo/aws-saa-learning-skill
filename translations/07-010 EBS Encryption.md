# EBS 加密

## 学习目标
- 理解 EBS 加密覆盖范围
- 掌握从未加密卷迁移到加密卷的方法

## 重点速览
- 使用 KMS（默认 AWS 托管密钥）
- 静态数据、快照、传输全程加密
- 可通过快照复制实现加密迁移

## 详细内容
EBS 加密的核心点：
- **默认 KMS 加密**：创建加密卷时选择 KMS 密钥
- **端到端加密**：卷数据、快照、卷间传输均加密
- **继承性**：
	- 加密卷 -> 快照自动加密
	- 加密快照 -> 新卷保持加密

从未加密卷迁移到加密卷：
1. 为未加密卷创建快照
2. 复制快照时启用加密
3. 用加密快照创建新卷

性能影响通常很小。
