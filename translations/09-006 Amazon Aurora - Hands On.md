# Aurora 实操

## 学习目标
- 创建 Aurora 集群
- 理解 writer/reader 端点
- 了解读副本与自动扩缩选项

## 重点速览
- 支持 MySQL/PostgreSQL 兼容版本
- 可选 Aurora Standard/IO-Optimized
- 可配置多 AZ 读副本与自动扩缩

## 详细内容
创建集群：
1. 选择 **Standard create**。
2. 选择 MySQL 或 PostgreSQL 兼容版本。
3. 选择模板（如 Production）。
4. 设置集群标识符、管理员账号与密码。

存储与实例：
- 存储选择 **Aurora Standard** 或 **IO-Optimized**。
- 选择实例类型（如 `db.t3.medium`）。
- 如使用 Serverless v2，改为设置 ACU 范围。

可用性与网络：
- 启用 Aurora 副本（多 AZ）提升可用性。
- 配置 VPC、安全组、公开访问与端口（MySQL 为 3306）。

创建后验证：
- 集群包含 **writer** 与 **reader** 实例。
- 使用 **writer endpoint** 写入，**reader endpoint** 读取。
- 可添加跨 Region 读副本与读副本自动扩缩策略。
