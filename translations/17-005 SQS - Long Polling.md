**学习目标**

- 理解 SQS 长轮询（Long Polling）的作用、配置方式与带来的性能/成本优势。

**重点速览**

- 长轮询允许消费者在队列为空时最多等待 20 秒以接收新消息，减少空轮询导致的 API 调用次数与延迟。
- 可在队列级别或单次请求级别启用（等待时间 1–20 秒）。

**详细内容**

- 原理：短轮询会立即返回当前可见消息，但会产生大量空响应与 API 调用；长轮询在队列为空时阻塞等待消息到达，若在等待期内有消息到来则立即返回，降低调用频率并提高效率。
- 优点：减少对 SQS 的 API 调用计数（节省成本）、降低延迟（消息一到即被推送返回）。
- 配置建议：优先在队列上设置较长的默认等待时间（例如 20 秒），或让消费者在调用 ReceiveMessage 时指定等待秒数；对于高吞吐场景与短延迟需求，综合评估等待时间与并发消费者数。

**自测问题**

1. 长轮询如何帮助降低对 SQS 的 API 调用次数？
2. 长轮询可以设置的最大等待时间是多少？
