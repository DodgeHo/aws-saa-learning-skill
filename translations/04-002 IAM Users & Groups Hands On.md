# IAM 用户与组实操

## 学习目标
- 会在 IAM 中创建用户与组
- 理解 IAM 是全球服务（无区域）
- 知道如何使用账号别名与 IAM 用户登录

## 重点速览
- IAM 是全球服务，右上角区域选择器不可用
- 建议创建管理员 IAM 用户替代 root 登录
- 权限通过组继承，便于集中管理

## 详细内容
进入 IAM 控制台后会看到安全建议。右上角区域选择器不可用，说明 IAM 是全球服务，用户在所有区域都可用。

建议不要长期使用 root 用户。我们创建一个管理员 IAM 用户（如 `Stephane`），选择“创建 IAM 用户”（考试重点），并设置登录密码。若为他人创建账号，可用系统生成密码并强制首次登录后修改；为自己创建可使用自定义密码。

接着创建一个组（如 `admin`），并为组附加 `AdministratorAccess` 策略。将用户加入该组即可继承管理员权限。可选地为用户添加标签（如部门）。创建完成后，可下载 CSV 或发送登录说明。

在用户与组列表中可看到权限继承关系：用户通过组获得管理员权限。通过组统一管理权限更简单、可扩展。

登录方面：在 AWS 账号页面可以获取登录 URL，并创建**账号别名**简化登录地址。建议使用一个正常窗口登录 root、私密窗口登录 IAM 用户，以便并行对照。IAM 用户登录需要输入账号 ID/别名、IAM 用户名与密码。

务必保管好 root 与管理员登录信息，否则恢复会非常麻烦。课程中多数操作建议使用 IAM 用户，只有少数操作需要 root 登录。
